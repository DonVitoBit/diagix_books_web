# Техническое задание на фронтенд приложения «Text Re-phraser»

## 1. Общие сведения

### 1.1 Назначение
Веб-приложение для интеллектуальной обработки научных текстов (перефразирование книг), управления переписанными книгами, доступа модераторов и автоматической иллюстрации на основе AI.

### 1.2 Цель ТЗ
Описание функциональных и интерфейсных требований к фронтенду для переноса текущего Streamlit-приложения на отдельный SPA (React / Vue / Svelte) с бэкендом на FastAPI.

### 1.3 Текущий стек (для справки)
- UI: Streamlit (Python)
- БД: SQLite
- Логика: Python (main.py, core/db.py, core/users.py, settings_manager, illustration_pipeline)

---

## 2. Роли пользователей и авторизация

### 2.1 Роли
| Роль        | Описание                                                                 |
|------------|--------------------------------------------------------------------------|
| **admin**  | Полный доступ: все разделы, настройки API, модераторы, все книги.        |
| **moderator** | Доступ только к разделу «Книги»; видит только книги, к которым ему выдан доступ. |
| **guest**  | Не авторизован — видна только страница входа.                           |

### 2.2 Авторизация
- **Страница входа:** логин + пароль.
- Администратор: логин/пароль из конфига (например, `admin` / `admin123`).
- Модератор: проверка по БД (логин + хеш пароля).
- После входа: хранение сессии (JWT или cookie), отображение в интерфейсе: «Пользователь: {имя} ({роль})», кнопка «Выйти».
- Подсказка на странице входа: «Логин и пароль задаются в config / выдаются администратором».

### 2.3 Навигация по ролям
- **admin:** Книги | Перефразирование | Результаты | Иллюстрации | Настройки | Модераторы.
- **moderator:** только «Книги» (доступные ему).
- **guest:** только форма входа.

---

## 3. Структура приложения (разделы)

1. **Вход** — форма логина (гость).
2. **Книги** — список книг, выбор книги, управление (admin) или просмотр/редактирование доступных (moderator).
3. **Перефразирование** — загрузка файла, настройки, запуск перефразирования, прогресс (только admin).
4. **Результаты** — результат из файла (последний/частичный) + список всех переписанных книг из БД (admin).
5. **Иллюстрации** — загрузка PDF, извлечение изображений, перерисовка/улучшение через AI (admin).
6. **Настройки** — API-ключи (OpenAI, DeepSeek, DALL-E, NanoBanana, Tavily, Google), провайдер LLM, параметры обработки, справка (admin).
7. **Модераторы** — создание модераторов, смена пароля, удаление, выдача доступа к книгам (admin).

## 4. Детальное описание экранов и функций

### 4.1 Вход (Login)
- Заголовок/бренд: «Text Rephraser», подзаголовок: «Интеллектуальная обработка научных текстов».
- Поля: **Логин** (текст), **Пароль** (тип password).
- Кнопка: **Войти** (primary).
- При успехе: редирект в раздел по роли (Книги или первый таб для admin).
- При ошибке: сообщение «Неверный логин или пароль».
- Футер: подсказка про логин/пароль (config или администратор).

### 4.2 Книги (Books)

#### 4.2.1 Для администратора
- Заголовок: «Книги».
- **Список книг:** выпадающий список (select) или список карточек: «#{id} — {title}». Выбор книги — показать детали выбранной.
- Мета по книге: дата создания, автор (created_by), исходный файл (source_filename).
- **Название:** поле ввода + кнопка «Сохранить название».
- **Скачать:** кнопка «Скачать переписанный текст» (файл .txt).
- **Доступ модераторов:** список модераторов с чекбоксами (кому выдан доступ). Кнопка «Сохранить доступ».
- **Просмотр:** раскрываемая область с текстовым полем (переписанный текст книги), только чтение, высота ~420px.
- **История версий:** список версий (номер, дата, автор, примечание). Для каждой версии: превью текста, кнопка «Восстановить эту версию».

#### 4.2.2 Для модератора
- Заголовок: «Книги».
- **Доступные вам книги:** список книг, к которым выдан доступ (выбор одной).
- По выбранной книге:
  - Просмотр переписанного текста (textarea, только чтение или редактирование).
  - **Редактирование:** текстовое поле переписанного текста + кнопка «Сохранить правки».
  - **Перегенерация абзацев:** текст разбит на блоки (по абзацам), чекбоксы для выбора блоков + кнопка «Перегенерировать выбранные» (вызов API перефразирования выбранных частей).
  - **Комментарии:** список комментариев (автор, текст, дата, опционально индекс абзаца); форма добавления комментария.
- **История версий:** как у admin — список версий, превью, «Восстановить эту версию».
- Скачать переписанный текст (кнопка).

### 4.3 Перефразирование (Paraphrasing)
- Заголовок: «Перефразирование текста».
- **Параметры:**
  - Тема текста (текст, обязательное).
  - Загрузка файла: PDF, TXT, MD, DOCX; лимит 10 MB.
  - Temperature (слайдер 0.0–1.0, по умолчанию 0.4).
  - Включить исследование (чекбокс include_research).
- Кнопка: **«Начать перефразирование»** (primary).
- После старта:
  - Прогресс: этапы (извлечение текста, перефразирование блоков и т.д.) и прогресс-бар.
  - Два столбца: «Сейчас в работе (оригинал)» — текущий блок; «Накапливаемый результат» — текст всех уже перефразированных блоков (textarea или read-only + markdown-превью).
  - По окончании: превью результата, кнопки «Скачать .md», «Скачать .pdf» (если доступно).
  - Кнопка «Перейти к результатам».
- При прерывании/ошибке: сообщение «Обработка прервана. Частичные результаты сохранены…»; частичный результат тоже сохраняется в БД и отображается на вкладке «Результаты».

### 4.4 Результаты (Results)
- Заголовок: «Результаты перефразирования».
- **Результат из файла** (если есть актуальный результат в output):
  - Подзаголовок: «Результат из файла» или «Результат из файла (частичный или последний)» при частичном результате.
  - Подпись: частичный результат — «Частичный результат перефразирования (обработка была прервана)…».
  - Два столбца: «Оригинальный текст» и «Перефразированный текст» (раскрываемые блоки, textarea только чтение, высота ~200px).
  - Кнопка **«Добавить в книги»**: сохранить текущий результат из файла как новую книгу в БД.
- **Все переписанные книги:**
  - Список книг из БД: для каждой — раскрываемая карточка «{title} — {дата} ({created_by})»; при частичном результате — пометка «— частичный результат».
  - Внутри карточки: предупреждение «Частичный результат…» при необходимости; два столбца — оригинальный и перефразированный текст (textarea только чтение); кнопка «Скачать перефразированный текст»; кнопка «Добавить в книги» (установить last_saved_book_id и т.п., если нужно поведение как в текущем приложении).

### 4.5 Иллюстрации (Images)
- Заголовок: «Автоматическая иллюстрация книги».
- **Загрузка PDF:** компонент загрузки файла (только PDF), отображение имени и размера.
- Чекбокс: «Извлечь изображения из загруженной книги».
- Кнопка **«Извлечь изображения»** — запуск извлечения; во время процесса — прогресс и превью последних извлечённых изображений (например, по 4 в ряд).
- После извлечения:
  - Сводка: всего изображений, по классам (clinical, encyclopedia и т.д.).
  - **Просмотр изображений:** навигация (предыдущее/следующее или селектор «N из M»); для текущего изображения — превью, подпись/метаданные.
  - **Перерисовка:** опция «Улучшить промпт с помощью AI», поле промпта, кнопки «Перерисовать заново», «Улучшить через NanaBanana Pro» и т.п.; отображение исходного и улучшенного изображения.
- Секция «Найденные изображения из поиска» (если API возвращает результаты поиска): галерея по патологиям/запросам.

### 4.6 Настройки (Settings)
- Заголовок: «Настройки и информация».
- **Управление API ключом (OpenAI):** статус (ключ сохранён / не установлен), маска ключа (первые/последние символы). Кнопки «Изменить», «Удалить», «Добавить». Форма: поле пароля для ключа, кнопки «Сохранить», «Проверить» (проверка ключа через API), «Отмена».
- **Выбор провайдера LLM:** переключатель (radio): OpenAI (ChatGPT) / DeepSeek.
- **DeepSeek API:** аналогично OpenAI — статус, изменить/удалить/добавить ключ, форма с сохранением/отменой.
- **API ключи для иллюстраций:** отдельные блоки для DALL-E 2, NanoBanana, Tavily, Google Custom Search (ключ + ID поисковика при необходимости). Для каждого: удалить/добавить/настроить, форма с сохранением/отменой.
- **Параметры обработки:** temperature (слайдер), описание диапазонов; при необходимости — другие параметры из config.
- **Инструкции по использованию:** текст справки (шаги работы с перефразированием).
- **Возможные проблемы:** раскрываемая секция «Распространенные ошибки и решения».

### 4.7 Модераторы (Moderators)
- Заголовок: «Модераторы».
- Доступ только для admin; иначе — сообщение «Доступ только для администратора».
- **Создать модератора:** форма: Имя, Логин (3–32 символа, латиница/цифры/._-), Пароль, Повторите пароль. Валидация: логин не admin, пароль не короче 6 символов, пароли совпадают. Кнопка «Создать».
- **Список модераторов:** для каждого — раскрываемая карточка: имя, логин, дата создания; форма «Сменить пароль» (новый пароль, повторить, «Сохранить пароль»); кнопка «Удалить модератора»; список книг с чекбоксами «Выдать доступ» и кнопка «Сохранить доступ».

## 5. Модели данных (для API)

Сущности, которые фронт получает/отправляет через API (форматы — JSON).

### 5.1 Книга (Book)
- `id` (int)
- `title` (string)
- `source_filename` (string | null)
- `theme` (string | null)
- `temperature` (number | null)
- `include_research` (boolean)
- `original_text` (string | null)
- `paraphrased_text` (string)
- `created_at` (ISO datetime string)
- `created_by` (string)

### 5.2 Версия книги (BookVersion)
- `id` (int)
- `book_id` (int)
- `version_number` (int)
- `paraphrased_text` (string)
- `change_note` (string | null)
- `created_at` (string)
- `created_by` (string)

### 5.3 Комментарий (Comment)
- `id` (int)
- `book_id` (int)
- `author` (string)
- `comment_text` (string)
- `paragraph_index` (int | null)
- `created_at` (string)

### 5.4 Модератор (Moderator)
- `username` (string)
- `name` (string | null)
- `created_at` (string)

### 5.5 Доступ модератора к книге
- `book_id` (int)
- `moderator_username` (string)
- Список выданных доступов по книге: массив `moderator_username`.

### 5.6 Результат из файла (текущий run)
- Читается с бэкенда или из отдельного эндпоинта: `original_text`, `paraphrased_text` (последний/частичный результат из output или сессии).

### 5.7 Настройки (для отображения/сохранения)
- Ключи API: только маскированное отображение и приём нового значения (не отдавать полный ключ в ответах).
- Провайдер LLM: `openai` | `deepseek`.
- Temperature, include_research и т.д. — по текущей схеме настроек.

## 6. Требования к UI/UX

- Адаптивная вёрстка: комфортная работа на десктопе и планшете.
- Единый стиль: шрифты (например, Inter), цветовая схема, скругления, тени — согласованы с текущим брендингом (например, градиенты, акцентные кнопки).
- Состояния: загрузка (спиннеры/скелетоны), пустые состояния (нет книг, нет модераторов), ошибки (сообщения под формами или тосты).
- Формы: валидация на клиенте + отображение ошибок с бэкенда.
- Длинные тексты: скролл, ограничение высоты textarea, при необходимости виртуализация списков.
- Доступность: семантичная разметка, контраст, фокус у кнопок и полей.

## 7. Технические требования к фронтенду

- SPA на выбранном стеке (React, Vue или Svelte + TypeScript рекомендуется).
- Роутинг по разделам: `/login`, `/books`, `/paraphrasing`, `/results`, `/images`, `/settings`, `/moderators` (или аналог).
- Управление состоянием: глобальное (пользователь, роль, выбранная книга) + локальное по страницам.
- Запросы к API: через единый HTTP-клиент (fetch/axios), базовый URL из конфига, обработка 401 (редирект на логин), 403 (доступ запрещён).
- Загрузка файлов: multipart/form-data; для длительного перефразирования — опционально SSE или polling для прогресса.
- Работа с большими текстами: не блокировать UI (стриминг/чанки при отображении при необходимости).

## 8. Интеграция с бэкендом (API)

Бэкенд должен предоставить REST (или GraphQL) API, например:

- **Auth:** POST `/auth/login` (login, password) → токен + роль + имя; POST `/auth/logout`; GET `/auth/me`.
- **Books:** GET `/books`, GET `/books/{id}`, PATCH `/books/{id}` (title, paraphrased_text), POST `/books` (создание из результата перефразирования).
- **Versions:** GET `/books/{id}/versions`, POST `/books/{id}/versions/restore` (version_id).
- **Access:** GET `/books/{id}/access`, PUT `/books/{id}/access` (список moderator_username).
- **Comments:** GET `/books/{id}/comments`, POST `/books/{id}/comments`.
- **Paraphrasing:** POST `/paraphrase` (file, theme, temperature, include_research) — синхронно или с job_id + GET `/paraphrase/status/{job_id}`; GET `/paraphrase/last-result` (original_text, paraphrased_text из файла/сессии).
- **Images:** POST `/images/extract` (PDF file), GET `/images/extracted`, POST `/images/redraw` и т.п. — по фактическим возможностям бэкенда.
- **Settings:** GET `/settings` (маскированные ключи, провайдер, temperature и т.д.), PATCH `/settings`.
- **Moderators:** GET `/moderators`, POST `/moderators`, DELETE `/moderators/{username}`, PATCH `/moderators/{username}/password`, управление доступом к книгам (через Books/Access или отдельные эндпоинты).

Детализацию эндпоинтов (тела запросов/ответов) целесообразно оформить отдельным документом (OpenAPI/Swagger) по мере реализации бэкенда.

## 9. Приложения

### 9.1 Константы из config (для справки)
- Поддерживаемые типы файлов: pdf, txt, md, docx.
- Макс. размер файла: 10 MB.
- Temperature: по умолчанию 0.4, диапазон 0.0–1.0.
- Названия табов: Книги, Перефразирование, Результаты, Иллюстрации, Настройки, Модераторы.

### 9.2 Глоссарий
- **Перефразирование** — обработка текста через LLM с сохранением смысла и академического стиля.
- **Частичный результат** — перефразирование прервано; сохранён только часть текста.
- **Версия книги** — снимок переписанного текста с примечанием и датой; можно восстановить.
