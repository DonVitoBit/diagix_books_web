# История изменений

## [Неопубликовано] — 2025-03

- **Документация** — папка `docs/` с подробной технической документацией:
  - [README](README.md) — оглавление и быстрый старт
  - [ARCHITECTURE](ARCHITECTURE.md) — структура проекта, поток данных
  - [INSTALLATION](INSTALLATION.md) — установка и запуск
  - [CONFIGURATION](CONFIGURATION.md) — настройки и API-ключи
  - [API_REFERENCE](API_REFERENCE.md) — справочник модулей и методов
  - [DEPLOYMENT](DEPLOYMENT.md) — развёртывание на сервере
  - [USER_GUIDE](USER_GUIDE.md) — руководство пользователя

### Изменено

#### Отображение статей (Markdown)

- **ui/tabs/text_tab.py**: рендеринг статей переведён с `st.markdown(..., unsafe_allow_html=True)` на `components.html()` для корректного отображения Markdown (заголовки, списки, **жирный**, цитаты) в динамически обновляемых контейнерах.

#### Иллюстрации

- **ui/tabs/text_tab.py**: при неудачной генерации иллюстрации вместо сообщения «[подпись — иллюстрация не сгенерирована]» выводится только подпись курсивом: *подпись*.

#### Лимит токенов для статей

- **main.py**: лимит `max_tokens_article` увеличен с 8192 до 65536 (по умолчанию 16384).
- **ui/tabs/settings_tab.py**: слайдер «Лимит токенов для статей» — диапазон 2048–65536, шаг 1024.
- **settings_manager.py**: значение по умолчанию `max_tokens_article` = 16384.

#### Кнопки скачивания

- **ui/tabs/text_tab.py**: кнопки «Скачать статью (.md)» и «Скачать статью (.pdf)» перенесены под поле «Тема статьи» (для режимов «по теме» и «по документам»). Отображаются после генерации, статья сохраняется в `session_state`.

#### Подбор статей PubMed

- **core/pubmed.py**:
  - `filter_entries_by_title_relevance`: проверка по границам слов (без ложных совпадений типа «hiv» в «shiv»), сортировка по релевантности.
  - `search_pubmed_ids`: fallback при 0 результатах — повторный поиск с первым словом темы (например, `HIV[tiab]` вместо `"HIV infection"[tiab]`).
  - Добавлена функция `score_entry_by_theme()` для оценки релевантности статьи.
- **main.py**:
  - `execute_article_searches`: `max_results` увеличен с 8 до 12, до 6 статей на запрос.
  - Итоговая сортировка статей по релевантности (больше слов темы в заголовке — выше в списке).

### Исправлено

- **main.py**: исправлены ошибки отступов (IndentationError) в `split_block`, `_paraphrase_one_chunk`, `paraphrase_block`, `process_text`, CLI.
- **app.py**: защита от `KeyError` при пустом `tab_map` (роль guest), проверка `"Книги" in tab_map` перед рендерингом табов.
